# OBS 非依存アーキテクチャ概要

## 目的

master-duel-recorder-lite は、OBS を使わずに Master Duel の録画支援を行うための Python アプリとして設計します。既存の OBS プラグイン方式は強力ですが、C++、OBS SDK、Qt、OBS 本体の互換性に依存します。このリポジトリでは、その依存を外して、Python で理解しやすく保守しやすい構成を優先します。

## 基本構成

| 領域 | 責務 |
|---|---|
| Capture | 対象ウィンドウや画面の取得方法を管理する |
| Recorder | FFmpeg などを使って録画ファイルを作る |
| Detection | 状態遷移、テンプレートマッチング、補助的なOCRを扱う |
| Storage | SQLite、設定、キュー状態を保存する |
| Upload | YouTube などへのアップロード準備と結果保存を扱う |
| UI | 設定と状態確認をユーザーに見せる |

初心者向けに言うと、録画する係、判定する係、保存する係、アップロードする係を分けます。分けておくと、録画方法だけを変更してもDBやアップロード処理まで壊しにくくなります。

## 実行時データ

ユーザーごとの設定、ログ、DB、録画キューは `user_data/` に保存します。このフォルダは Git にコミットしません。アプリ本体と実行時データを分けることで、アップデート時にユーザーの録画履歴や認証情報を守りやすくします。

## 録画方式の考え方

初期方針では、Python が録画対象を決め、FFmpeg がエンコードを担当します。Python は制御係、FFmpeg は動画作成係です。これにより、重い動画処理を Python の中で無理に抱え込まずに済みます。

## 注意点

音声キャプチャ、GPUエンコード、対象ウィンドウの安定取得は環境依存が強い領域です。そのため、最初から完全自動化を目指さず、短い録画検証、保存先確認、復旧処理の順に実装します。
